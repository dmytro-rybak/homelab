# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Personal homelab Kubernetes cluster — **Talos Linux** running directly on **Hetzner Cloud** (3 CPX22 control planes, one per eu-central location) plus **one bare-metal worker** on an existing Hetzner dedicated server. Cilium is the CNI (replaces kube-proxy, transparent WireGuard encryption). Cluster is reachable only via WireGuard. Everything is provisioned by **OpenTofu** end-to-end — no Proxmox, no Ansible, no GitOps tooling.

## Required Tools (macOS)

```bash
brew install talosctl kubectl helm cilium-cli opentofu wireguard-tools
```

## Architecture

- **Cloud control planes**: 3 × CPX22 in nbg1, fsn1, hel1; boot the public Hetzner Talos ISO; receive machine config via Hetzner metadata user_data on first boot.
- **Bare-metal worker**: existing dedicated server (Talos installed by `dd`-ing the metal image to disk via the Hetzner rescue system). OpenTofu pushes its config via `talos_machine_configuration_apply` against the existing public IP.
- **Hetzner Cloud private network** (`10.20.0.0/16`) with two subnets:
  - cloud (`10.20.0.0/24`) — control planes
  - vswitch (`10.20.1.0/24`) — couples a Robot vSwitch (created manually) to the cloud network so the dedicated worker is reachable on the same private fabric. MTU is 1400 across this path.
- **WireGuard** (`10.25.0.0/24`) — host-level WG interface (`wg0`) on every node. Only peer is the laptop. Keys are generated by OpenTofu (`wireguard_asymmetric_key`); the laptop's public key is passed in via tfvars.
- **kube-proxy** is disabled in Talos; **Cilium** replaces it. **KubePrism** is explicitly enabled on every node (kubelet → `localhost:7445` → CP API servers).
- **Cluster endpoint**: `api.internal.homelab0.xyz` — Cloudflare A records point to all three CP public IPs (managed in TF). Public ingress on each node is locked down to UDP 51820 (WG) after bootstrap; `bootstrap_complete` tfvar gates this transition.
- **Cilium**: tunnel mode (vxlan), MTU 1400, transparent WireGuard node encryption, Hubble + Hubble UI, Gateway API CRDs (no Gateway resource yet), LB IP pool defined but `disabled: true`.
- **cert-manager**: Let's Encrypt via Cloudflare DNS-01 (staging + prod ClusterIssuers). Wildcard certificate covering `*.internal.homelab0.xyz` is requested as a Kubernetes Secret. No HTTP-01 — services are not publicly exposed.
- **external-dns**: Cloudflare provider, watches Service + Gateway HTTPRoute, scoped to `internal.homelab0.xyz`, records are DNS-only (not proxied — pointing at private IPs).

## Bringing the cluster up from scratch

```bash
export HCLOUD_TOKEN=...                      # Hetzner Cloud
export CLOUDFLARE_API_TOKEN=...              # Cloudflare (Zone:DNS:Edit on homelab0.xyz)
export TF_VAR_cloudflare_api_token=$CLOUDFLARE_API_TOKEN

# Manual prep (one-time):
# 1. Create vSwitch in Robot, attach the dedicated server, note the numeric vswitch_id.
# 2. Boot the dedicated server into rescue, dd Talos metal image to /dev/nvme0n1, reboot.
#    Server comes up in maintenance mode on its public IP, port 50000.
# 3. Set vswitch_id in opentofu/infrastructure/terraform.tfvars.

cd opentofu/infrastructure
tofu init -backend-config=backend.hcl && tofu apply

# After all resources are healthy:
tofu output -raw kubeconfig            > ~/.kube/homelab.config
tofu output -raw talosconfig           > ~/.talos/config
tofu output -raw wireguard_laptop_config > ~/.config/wireguard/homelab.conf
# edit homelab.conf and replace REPLACE_WITH_LAPTOP_PRIVATE_KEY

# Bring up the WG tunnel, verify nodes, then lock down the firewall:
sudo wg-quick up ~/.config/wireguard/homelab.conf
KUBECONFIG=~/.kube/homelab.config kubectl get nodes
# Once everything is Ready: flip bootstrap_complete = true in tfvars, tofu apply again.
```

## Secrets & state

- `*.tfstate`, `.terraform/`, kubeconfigs, certs (`.pem`/`.key`/`.crt`), and all `wg*.conf` are gitignored.
- The Cloudflare API token is the only externally-provided secret; everything else (Talos PKI, WG keypairs, etc.) is generated by TF and lives in state. Treat the state file as sensitive.
- State backend: S3-compatible Hetzner Object Storage (`backend.hcl` is gitignored).

Do not use WebFetch, WebSearch, or rely on training data for library docs.

Before writing code or configuration that uses a library or provider, query its docs via `context` to confirm the current API.

## Security Notes

- WireGuard private/public keys and `wg*.conf` files are gitignored — never commit them
- Kubeconfig and all credential files (`.pem`, `.key`, `.crt`, `.env`) are gitignored

## Golden Rule

**Always prefix commands with `rtk`**. If RTK has a dedicated filter, it uses it. If not, it passes through unchanged. This means RTK is always safe to use.

**Important**: Even in command chains with `&&`, use `rtk`:
```bash

# ✅ Correct
```
